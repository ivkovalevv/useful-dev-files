## 1. Структура запроса в SQL

Запрос на выгрузку данных начинают со слова SELECT (англ. «выбрать»). Всё, что мы напишем после этого слова, отобразится в итоговом результате.
```SQL
SELECT 'Здравствуйте, я ваш первый запрос!';
```
| ?column? |
| :--- |
| Здравствуйте, я ваш первый запрос! |

<br><br/>
При помощи SELECT можно решать арифметические примеры: например, к восьми прибавить два и результат разделить на пять. 
При этом сохраняется приоритет арифметических операторов. Первыми выполняют умножение и деление, затем — сложение и вычитание. 
Скобки имеют наивысший приоритет.
```SQL
SELECT (8+2)/5;
```
| ?column? |
| :--- |
| 2 |

<br><br/>
Чтобы выгрузить данные из таблицы, нужно указать, откуда именно их брать. 
Для этого дополним запрос словом `FROM` (англ. «из»).

Запрос выглядит так:
```SQL
SELECT *
FROM buyer;
```

После слова `FROM` указывают таблицу, из которой нужно выгрузить данные. 
В нашем случае это будет таблица `buyer`. Получаем запрос «выбрать всё из таблицы buyer».  
Результатом будет выгрузка всей таблицы `buyer`:

| :--- | :---: | ---: | ---: | ---: | ---: | ---: | ---: |
| 145734 | Коновалова | Ольга | Женский | 46 | Роботический лабиринт | 4 | 0 |
| 145734 | Тарасова | Ирина | Женский | 22 | Фантастическая фабрика | 4 | 0 |
| 145734 | Воробьев | Артем | Мужской | 25 | Роботические гонки | 3 | 0 |
| 145734 | Матвеев | Виктор | Мужской | 26 | Интерактивный музей робототехники | 2 | 0 |
| … | … | … | … | … | … | … | … |

<br><br/>
Подсчитать число записей в таблице можно, поместив символ * в оператор COUNT (англ. «посчитать»):
```SQL
SELECT COUNT(*)
FROM buyer;
```
| count |
| :--- |
| 178 |

<br><br/>
Запрос к базе данных может выглядеть так:
```SQL
select sp.advertising_id,sp.install_date,sp.session_num,sp.payer,sp.last_active as last_date, DATE_DIFF(sp.last_active, install_date, day) as max_play from players as sp where sp.date = '2021-02-28' and sp.install_date between '2021-02-01' and '2021-02-26';
```

А так выглядит тот же запрос, написанный по правилам:
```SQL
SELECT sp.advertising_id,
       sp.install_date,
       sp.session_num,
       sp.payer,
       sp.last_active AS last_date,
       DATE_DIFF(sp.last_active, install_date, DAY) AS max_play
FROM players AS sp
WHERE sp.date = '2021-02-28'
  AND sp.install_date BETWEEN '2021-02-01' AND '2021-02-26';
```

## 2. Фильтрация данных

Для отбора данных и нужна фильтрация.
Например, нужно определить, как праздники влияют на продажи. Специалист может отобрать данные за предновогодний декабрь и за нейтральный месяц — например, октябрь. А затем сравнить динамику выручки в эти два месяца по отдельности и проанализировать, насколько выручка отличается.
<br><br/>
### 2.1. Ограничение выгрузки с помощью `SELECT`

Так можно выгрузить всю информацию из таблицы:
```SQL
SELECT *
FROM buyer;
```

Так можно выгрузить только три поля — last_name, first_name, gender:
```SQL
SELECT last_name,
       first_name,
       gender
FROM buyer;
```
В каком порядке поля перечислены в запросе, в таком они отобразятся в итоговой таблице.

| last_name | gender |first_name |
| :--- | :---: | ---: |
| Коновалова | Женский | Ольга |
| Тарасова | Женский | Ирина |
| Воробьёв | Мужской | Артём |
| … | … | … |

<br><br/>
### 2.2. Переименование поля с помощью `AS`

Каждому полю в запросе можно назначить псевдоним, или алиас. 
Для этого после нужного поля указывают оператор `AS` (англ. «как») и пишут псевдоним.
```SQL
SELECT first_name,
       age AS age_client,
       connection_area AS area
FROM buyer;
```

| first_name | age_client | area |
| :--- | :---: | ---: |
| Ольга | 46 | Роботический лабиринт |
| Ирина | 22 | Фантастическая фабрика |
| Артём | 25 | Роботические гонки |
| Виктор | 28 | Интерактивный музей робототехники |
| … | … | … |

<br><br/>
### 2.3. Ограничение выгрузки строк с помощью `LIMIT` и `OFFSET`

Оператор `LIMIT` ставят в конце запроса и после него указывают количество записей (строк), которые нужно выгрузить.
Такой запрос отобразит десять записей из перечисленных полей:
```SQL
SELECT поле_1,
       поле_2,
       поле_3
             ...
FROM таблица
LIMIT 10;
```
<br><br/>
Оператор `OFFSET` позволяет пропустить определённое число записей.
Если нужно указать, с какой записи начинать выборку, к оператору `LIMIT` добавляют оператор `OFFSET`.
Такой запрос выведет записи с 6 по 15:
```SQL
SELECT поле_1,
       поле_2,
       поле_3
             ...
FROM таблица
LIMIT 10 OFFSET 5;
```
<br><br/>
`OFFSET` — самостоятельный оператор. Чтобы отобразить записи всей таблицы, начиная с определённой, указывают только оператор `OFFSET` без оператора `LIMIT`. 
Например, если нужно отобразить всю таблицу, начиная с шестой записи, запрос будет выглядеть так:
```SQL
SELECT поле_1,
       поле_2,
       поле_3
             ...
FROM таблица
OFFSET 5;
```
<br><br/>
Пример выгрузки только поля last_name, gender, age с 7 по 10 строку включительно:
```SQL
SELECT last_name,
       gender,
       age
FROM buyer
LIMIT 4 OFFSET 6;
```

## 3. Условия выборки

### 3.1. Оператор `WHERE`

Часто данные нужно отфильтровать: выгрузить информацию за конкретный период или с конкретным условием. Например, получить строки, где количество купленного товара превышает 4 штуки. 
Для таких задач в SQL используют условный оператор `WHERE`. Его указывают после оператора `FROM`:
```SQL
SELECT поле_1, -- поля для выгрузки
       поле_2 
             ...
FROM таблица -- таблица, из которой выгружают данные
WHERE поле > 4; -- условие для среза данных
```

Задать условия можно с помощью операторов сравнения.  
Они выглядят так:  
`=` — равно;  
`!=` — не равно;  
`>` — больше;  
`<` — меньше;  
`>=` — больше или равно;  
`<=` — меньше или равно.

Сравнение с числом записывают просто: `WHERE поле < 5`.  
Но если значение сравнивают с символьным типом, набор символов нужно взять в одинарные кавычки: `WHERE поле = 'Иванов'`.   
Это правило также касается даты и времени: `WHERE поле = '2013-07-01'`.

<br><br/>
Пример выгрузки информации о клиентах, возраст которых превышает 50 лет:
```SQL
SELECT *
FROM buyer
WHERE age > 50;
```

В PostgreSQL в операторе `WHERE` нельзя использовать псевдонимы.

<br><br/>
### 3.2. Операторы `AND` и `OR`

Операторы `AND` и `OR` позволяют объединять условия.  
`AND` применяют в том случае, если выгружаемые данные должны соответствовать всем перечисленным условиям.  
Пример:
```SQL
WHERE connection_area = 'Роботический лабиринт'
  AND age > 30;
```
Клиенты младше 30 лет в список не войдут, так же как и подключившиеся в других зонах. 

<br><br/>
Отобрать данные, которые удовлетворяют одному из условий, можно оператором `OR`:
```SQL
WHERE age > 30 
  OR age < 20;
```
В таком списке окажутся не только клиенты старше 30, но и те, кто младше 20 лет. 

<br><br/>
### 3.3. Оператор `NOT`
Чтобы отобрать клиентов из всех зон, кроме зоны «Роботический лабиринт», можно «перевернуть» условие оператором `NOT`.  
С его помощью выгружают записи, которые не соответствуют условию:

```SQL
WHERE NOT connection_area = 'Роботический лабиринт'
```

Оператор `NOT` используют и в нескольких условиях.  
В результате такого запроса следом за тридцатилетними из выборки пропадут и двадцатилетние:
```SQL
WHERE NOT age = 30
  AND NOT age = 20;
```

<br><br/>
### 3.4. Оператор `IN`

Если условий, которые нужно перечислить в операторе `OR`, очень много, то код получится громоздким.  
Например, нужны все пользователи с именами Виктор, Любовь, Борис, Станислав, Алина, Евгения и Ольга старше 25 лет.  
Если использовать операторы `OR` и `AND`, то код выйдет таким:
```SQL
WHERE age > 25 
  AND (first_name = 'Виктор'
       OR first_name = 'Любовь'
       OR first_name = 'Борис'
       OR first_name = 'Станислав'
       OR first_name = 'Алина'
       OR first_name = 'Евгения'
       OR first_name = 'Ольга');
```

Этот код выдаст верный результат, но чтобы много раз не перечислять один и тот же оператор, можно просто использовать оператор `IN`:
```SQL
WHERE age > 25 
  AND first_name IN ('Виктор', 'Любовь', 'Борис', 'Станислав', 'Алина', 'Евгения', 'Ольга');
```

<br><br/>
### Приоритет операторов

Чем больше условий, тем сложнее уследить за логикой операторов. Условия с разными логическими операторами имеют разный приоритет.  
Первым всегда выполняется условие с оператором `NOT`, за ним — `AND` и в последнюю очередь — условие с `OR`. 

Управлять порядком можно с помощью скобок: арифметические операции в скобках выполняются первыми.  
В SQL так же: скобки помогут комбинировать условия и получать нужную выборку.
Пример:
```SQL
WHERE age > 30
  AND (connection_area = 'Роботический лабиринт'
  OR connection_area = 'Интерактивный музей робототехники');
```
Условие в скобках приоритетнее — значит, в таблицу попадут первые посетители двух зон, чей возраст больше 30 лет.

### 4.2. Функция `DATE_TRUNC`

Дата и время часто хранятся в таком виде: `'2009-11-19 11:03:05'`. Но сравнивать даты в таком формате неудобно, если нужен, например, только год.  
В таких случаях в PostgreSQL используют функцию `DATE_TRUNC`, которая «усекает» дату и время до необходимого значения.  
Синтаксис функции такой: `DATE_TRUNC('отрезок времени', поле)`.  

Отрезок времени может быть разным, главное — не забыть одинарные кавычки:  
'microseconds' — микросекунды;  
'milliseconds' — миллисекунды;  
'second' — секунда;  
'minute' — минута;  
'hour' — час;  
'day' — день;  
'week' — неделя;  
'month' — месяц;  
'quarter' — квартал;  
'year' — год;  
'decade' — десятилетие;  
'century' — век.  

Пример работы функции `DATE_TRUNC` с выделением месяца:
```SQL
SELECT date,
       DATE_TRUNC('month', date)::date
FROM hotdog
LIMIT 5;
```  

`::date` выведет дату без часовых поясов

| date | date_trunc |
| :--- | :---: |
| 2022-02-17 | 2022-02-01 |
| 2022-02-22 | 2022-02-01 |
| 2022-03-04 | 2022-03-01 |
| 2022-03-29 | 2022-03-01 |
| 2022-01-07 | 2022-01-01 |  

Теперь каждой дате соответствует первый день месяца, к которому принадлежит эта дата.
<br><br/>

По отдельному элементу даты можно проводить фильтрацию:  
```SQL
SELECT order_id,
       bracelet_id,
       date,
       name_hotdog,
       ingredients
FROM hotdog
WHERE DATE_TRUNC('month', date)::date = '2022-02-01'
LIMIT 5;
```

### 4.3. Функция `EXTRACT`

Чтобы получить только конкретную часть даты — год, месяц или минуту, — используют функцию `EXTRACT`.  
Её синтаксис: `EXTRACT(отрезок времени FROM поле)`.  

Одинарные кавычки для значений внутри `EXTRACT` не нужны. Отрезок времени можно представить следующими значениями:
CENTURY — век;  
YEAR — год;  
QUARTER — квартал;  
MONTH — месяц;  
WEEK — неделя в году;  
DAY — день;  
DOY (от англ. day of the year) — день года, выраженный числом от 1 до 365 или 366, если год високосный;  
DOW (от англ. day of the week) — день недели, выраженный числом от 0 до 6, где понедельник — 1, воскресенье — 0.  
ISODOW (от англ. day of the week и ISO 8601) — день недели, выраженный числом от 1 до 7, где понедельник — 1, воскресенье — 7;  
HOUR — час;  
MINUTE — минута;  
SECOND — секунда;  
MILLISECOND — миллисекунда.  

Выделим номер недели из поля с датой так же, как ранее мы делали с месяцем. На этот раз применим `EXTRACT`:  
```SQL
SELECT date,
       EXTRACT(WEEK FROM date)
FROM hotdog
LIMIT 5;
```

| date | extract  |
| :--- | ---: |
| 2022-02-17 | 7 |
| 2022-02-22 | 8 |
| 2022-03-04 | 9 |
| 2022-03-29 | 13 |
| 2022-01-07 | 1 |

Теперь напротив каждой даты появилось число — но уже не дата, а номер недели.

## 5. Вычисления

### 5.1. Создаём расчётное поле

Чтобы провести вычисления с числами, их напрямую прописывают в SELECT. Но часто вычисления нужно сделать для всего поля таблицы.  
Тогда создают новое поле, в котором отображается результат вычислений. Такое поле называют вычисляемым или расчётным.

Пример запроса для вывода таблицы с указанием скидки в долях:
```SQL
SELECT last_name,
       percent_of_discount,
       percent_of_discount/100.0 AS discount
FROM buyer
LIMIT 5;
```
Если не указать псевдоним, новое поле будет называться `?column?`.  
Один из элементов деления обязательно должен быть дробным типом, чтобы всё прошло как надо. Именно поэтому в запросе выше написано 100.0, а не 100.  
Если оба значения целочисленные, деление произойдёт без остатка.

| last_name | percent_of_discount | discount |
| :--- | :---: | ---: |
| Коновалова | 4 | 0.04 |
| Тарасова | 4 | 0.04 |
| Воробьев | 3 | 0.03 |
| Матвеев | 2 | 0.02 |
| Михайлов | 20 | 0.2 |

### 5.2. Агрегирующие функции

Агрегирующие функции позволяют проводить расчёты над группами данных и отображают результат в виде одной строки.  
Основные агрегирующие функции в SQL:  
`SUM(поле)` возвращает сумму значений в поле;  
`AVG(поле)` находит среднее арифметическое для значений в поле;  
`MIN(поле)` возвращает минимальное значение в поле;  
`МАХ(поле)` возвращает максимальное значение в поле;  
`COUNT(поле)` выводит количество записей в поле.  

Пример применения этих функций к полю `age` таблицы `buyer`:  
```SQL
SELECT SUM(age),
       AVG(age),
       MIN(age),
       MAX(age),
       COUNT(age)
FROM buyer;
```
Функция `SUM` сложила все значения, которые были в поле `age`.  
Функция `AVG` подсчитала среднее арифметическое всех значений, а функции `MIN` и `MAX` нашли минимум и максимум.  
Функция `COUNT` подсчитала все записи в поле.  

| sum | avg | min | max | count |
| :--- | :---: | :---: | :---: | ---: |
| 5477 | 30.7696629213483146 | 17 | 100 | 178 |  

Для агрегирующих функций особенности деления целочисленных типов данных не важны — все операции проходят одинаково: и для целочисленных типов, и для дробных.  

<br><br/>

### 5.3. Функция `DISTINCT`

Позволяет находить и рассчитывать показатели только по уникальным значениям.  

Пример выгрузки всех уникальных значений в поле `percent_of_discount`:  
```SQL
SELECT DISTINCT(percent_of_discount)
FROM buyer;
```

## 6. Группировка данных по категориям

Группировка — это расчёт агрегирующей функции не по всем данным, а по отдельным категориям.  

### 6.1. Оператор `GROUP BY`  

Пример расчета среднего возраста для покупателей обоих полов:  
сгруппируем данные по полю `gender` и подсчитаем среднее значение поля `age`.  
Для группировки применяют оператор `GROUP BY`: его записывают после `FROM`, если условия нет, и после `WHERE` — если оно есть.

```SQL
SELECT gender,
       AVG(age)
FROM buyer
GROUP BY gender;
```  

| gender | avg |
| :---: | :---: |
| Мужской | 31.4743589743589744 |
| Женский | 30.2200000000000000 |  

Группировка оператором `GROUP BY` устроена так: сначала группируют данные по одной категории, для неё вычисляют среднее, результат выводится на экран.   
И так — для каждой категории. Похоже на фильтрации, только запросов меньше. 

<br><br/>

### 6.2. Группировка с агрегацией и без

Группировку можно комбинировать с агрегирующими функциями и фильтрацией.  
Усложним задачу — посчитаем средний возраст клиентов каждого пола для тех, у кого скидка больше 3%:

```SQL
SELECT gender,
       AVG(age)
FROM buyer
WHERE percent_of_discount > 3
GROUP BY gender;
```  

| gender | avg |
| :---: | :---: |
| Мужской | 29.5925925925925926 |
| Женский | 30.6428571428571429 | 

<br><br/>

Если использовать оператор `GROUP BY` без агрегирующей функции, можно получить только список уникальных значений, которые содержатся в поле.  

```SQL
SELECT name_hotdog,
FROM hotdog
GROUP BY name_hotdog;
```  

| name_hotdog |
| :---: |
| Итальянская Дива |
| Азиатская Симфония |
| Пикантный Полумесяц |
| Вегетарианский Праздник |
| Гурманский Гриль |
| Мексиканская Фиеста |
| Французский Эклер |
| Карибский Потоп |
| Классический Взрыв Вкуса |
| Здоровый Зефир |

<br><br/>

### Важно! если поле указано в `SELECT`, оно обязательно должно присутствовать и в группировке.  Также, если в `SELECT` указаны категория и агрегация — значит, нужен оператор `GROUP BY`.

<br><br/>

В группировках также можно указывать псевдонимы. Подсчет средней цены товара для каждого месяца:  
```SQL
SELECT EXTRACT(MONTH FROM date) AS order_month,
       AVG(price)
FROM hotdog
GROUP BY order_month;
```

<br><br/>

Пример выполнения следующей задачи:  
Отобразите, сколько раз в среднем поливали горчицей каждый вид хот-дога, в котором пять ингредиентов.   
Добавьте условие: отберите записи за вторую неделю года.  

```SQL
SELECT  name_hotdog,
        AVG(mustard)

WHERE ingredients = 5
```

## 7. Фильтрация данных после группировки

`WHERE` работает только с исходными данными и не умеет работать с результатом вычислений, в связи с этим для фильтрации данных после группировки используются оператор `HAVING`.  

Он фильтрует значения, полученные в результате группировки — то есть работает не с новыми полями, а с результатом вычислений.  
Его записывают после `GROUP BY`.  

Пример:
```SQL
SELECT connection_area,
       AVG(age)
FROM buyer
WHERE company_marker = 0
GROUP BY connection_area
HAVING AVG(age) > 30;
```
Задача звучала бы так: подсчитай среднее по каждой зоне подключения, но только для клиентов, которые не являются сотрудниками парка. В итоговой таблице оставь только те результаты, где средний возраст больше тридцати лет.

| connection_area | avg |
| :---: | :---: |
| Футуристический цирк | 33.3750000000000000 |
| Фантастическая фабрика | 30.7741935483870968 |
| Роботический лабиринт | 30.7272727272727273 |
| Интерактивный музей робототехники | 31.9000000000000000 |

<br><br/>

### Особенности оператора `HAVING`

1. Условие, по которому оператор `HAVING` фильтрует данные, необязательно указывать в `SELECT`.  
2. Все операторы можно комбинировать с `HAVING` — но только при условии, что в запросе есть группировка `GROUP BY`. Без неё `HAVING` работать не будет.
3. `HAVING` может использовать агрегатные функции (`SUM`, `COUNT`, `AVG`, `MIN`, `MAX`) над любыми полями таблицы, даже если эти поля не включены в `SELECT`.
